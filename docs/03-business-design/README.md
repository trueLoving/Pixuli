## 编写时建议结构

新文档按**业务场景**组织，每篇建议包含：

1. **业务范围**：场景定义、参与角色、边界（在范围内 / 不在范围内）
2. **业务实现**：主流程、状态与规则、与 `@pixuli/core`
   / 插件 / 各端 UI 的衔接要点
3. **其他**：异常与错误、与 PRS 需求 ID 的映射、相关系统设计链接

具体篇目与序号将在 PRS 稳定后确定。
